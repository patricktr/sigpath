Guiding principles

  • Local-first & free. Every competitor is cloud-only or locked to a Windows/Visio/SQL stack. Native, offline, files-you-own is the wedge.
  • Don't fight the catalog moat. The incumbents license 1.5M-product databases. We win on a great custom-device builder + a community library + import — not catalog scale.
  • Port-first & "smart." Connections attach to typed ports, so the app can validate signal flow — the payoff the big tools charge thousands for.
  • Export is the field's soft spot. No indie tool generates a real BOM or cable schedule. Make rich auto-docs free.

Roadmap

Governing logic: you can't place gear without gear; you can't auto-generate docs without a complete model; and undo/persistence are miserable to retrofit — so they go early. free = nearly free from React Flow · new = added beyond the original list.

Phase 0

Canvas foundation

Done
  • Tauri v2 + React + TS scaffold
    Native Apple Silicon; public repo; verified build.
  • React Flow canvas with port-based device nodes
    Ports as typed handles; sample signal chain renders.
Phase 1

Make the editor real — build a diagram and keep it

In progress
  • Device / port / cable schema new
    The spine: controlled vocabularies (signals, connectors, cable types), device model vs. placed instance, compatibility primitive. Building now.
  • Local persistence — save/load diagrams to disk new
    Nothing is real until diagrams survive a restart. Files you own.
  • Custom item builder
    Define a device + inputs/outputs/labels. The on-ramp before the community DB exists.
  • Cable types + colors
    Typed, colored cables — the model everything downstream needs.
  • Undo / redo
    Build the history layer early or suffer later.
  • Snap-to-grid · multi-select · drag-and-drop free
    Mostly React Flow props/built-ins.
Phase 2

Organize & navigate

Planned
  • Folders / projects
    A project holds multiple diagrams. Builds on persistence.
  • Zones + labels
    Color-coded, labeled regions (stage, rack, control room).
  • Headings / lists / formatted-text annotations
  • Legend (cable-type color key)
    Trivial once cable types exist.
  • Left-to-right mode · zoom-to-fit free · zoom-to-zone
    fit is free via fitView; zoom-to-zone needs zones.
  • Light / dark mode free
    React Flow colorMode; can slide earlier anytime.
Phase 3

Documentation & export — the payoff

Planned
  • Auto packlists & patch lists
    Derive from the complete model. Make patch lists free (H2R gates them).
  • Export — diagram → PNG/JPG/PDF · lists → CSV/structured
Phase 4

Community equipment database

Planned
  • Grab from community library
    Bundled SQLite snapshot synced from a zero-egress CDN.
  • Community-updatable (contribution workflow)
    Low-volume write path (PR-against-data-repo or moderation queue). Schema designed for this in Phase 1.

Differentiators to add

Beyond the original list — each hits a competitor weakness. Rough priority order.

1Live signal validation

Warn on incompatible types (HDMI-out → SDI-in), dangling ports, over-subscribed outputs, duplicate cable IDs.

↳ ConnectCAD's $3.4k/yr Circuit Check — our moat, free

2Cable schedule + BOM export

Port-to-port table (ID, type, length, from→to) and equipment-quantity list.

↳ No indie tool has this

3Cable IDs + lengths + auto-numbering

ConnectCAD-style rules (prefix/source/dest). Lengths feed packlists & schedules.

↳ Table stakes for pros

4Rack elevation view

A second view (19" front/rear) from the same devices.

↳ Stardraw's "one model → many views"

5Room templates

Huddle room, classroom, streaming rig — fast starts.

6Printable cable labels

Brother/Dymo-style label sheets.

↳ Indie tools lack it

7Revision history

Natural with local files; SimpleWires charges for it.

8North-star: "describe a room → starter diagram"

XTEN-AV's AI frontier. Later, atop the community DB — but don't preclude it in the data model.
