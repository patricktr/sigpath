Guiding principles

  • Local-first & free. Every competitor is cloud-only or locked to a Windows/Visio/SQL stack. Native, offline, files-you-own is the wedge.
  • Don't fight the catalog moat. The incumbents license 1.5M-product databases. We win on a great custom-device builder + a community library + import — not catalog scale.
  • Port-first & "smart." Connections attach to typed ports, so the app can validate signal flow — the payoff the big tools charge thousands for.
  • Export is the field's soft spot. No indie tool generates a real BOM or cable schedule. Make rich auto-docs free.

Editing

Make the editor real — build a diagram and keep it
  • Device / port / cable schema new Hard
    The spine: controlled vocabularies (signals, connectors, cable types), device model vs. placed instance, compatibility primitive. Building now.
  • Local persistence — save/load diagrams to disk new Medium
    Nothing is real until diagrams survive a restart. Files you own.
  • Custom item builder Medium
    Define a device + inputs/outputs/labels. The on-ramp before the community DB exists.
  • Cable types + colors Easy
    Typed, colored cables — the model everything downstream needs.
  • Cables that are the converter new Hard
    A cable can carry different connectors on each end (e.g. 1/4" TRS ↔ 3.5mm TRS) as a single cable type — not an adaptor that is its own device. Today a 1/4"→3.5mm link surfaces a "needs adaptor" warning even when a matching cable exists. Model end-to-end connectors per cable so the validator only warns when no cable/adaptor can bridge the two ports. Note: asymmetric cables break the one-connector-per-cable model — a converter cable should render as a smooth color transition from connector A's color to connector B's color (gradient along the line) instead of a single flat cable color.
  • Live signal validation new Hard
    Warn on incompatible types (HDMI-out → SDI-in), dangling ports, over-subscribed outputs. The "port-first & smart" payoff, built on the schema's compatibility primitive. Mostly done. (Duplicate-cable-ID checking lives with the cable-ID feature, since it needs IDs to exist.) ↳ ConnectCAD's $3.4k/yr Circuit Check — our moat, free.
  • Adapter & converter awareness new Hard
    Name the transition cable when a run changes connector but not signal (HDMI→Mini-HDMI, ¼"→3.5mm) and list them in one spot; AC↔DC is the device's PSU. True signal mismatches (SDI→HDMI) are flagged as errors. ↳ no indie tool does cable-level adapter awareness.
  • Auto-add-converter button new Medium
    On a true signal-mismatch error, a one-click fix drops in a converter device (a learned default, a searchable picker, or — when the library has none — a pre-filled create wizard) and re-wires like-to-like on both sides. Builds on adapter/converter awareness flagging the mismatch.
  • Signal grade vs cable bandwidth — connector isn't enough new Hard
    Connector type (BNC) proves a cable can physically mate, not that it can carry the signal: a 3G-SDI cable won't pass a 6G/12G-SDI signal though both are BNC. Same shape everywhere — HDMI 1.4 / 2.0 / 2.1, USB 2.0 / 3.x / USB4, Cat5e vs Cat6a for 10GbE, DisplayPort versions. Add a bandwidth/grade rating on signals (required) and on cable types (supported), and grow the compatibility primitive from "connectors match" to "connectors match and the cable's rating ≥ the signal's demand." Ripples into catalog data — every cable/port needs a grade — so it's a schema + validation + data-entry change.
  • Undo / redo Hard
    Build the history layer early or suffer later.
  • Snap-to-grid · multi-select · drag-and-drop free Easy
    Mostly React Flow props/built-ins.
  • New devices spawn in the viewport center new Easy
    Place a newly added device at the center of the current view (snapped to grid) instead of a fixed/default spot. The cursor sits on the device picker when you add, so center-of-view — not the mouse — is the anchor that lands it where you're looking.
  • Patch multiple cables at once new Medium
    Bulk-wire a fan-out: select source ports 1–8 on the video router, then destination ports 1–8 on the ATEM, and run all eight cables in order in one action — instead of dragging each link by hand. Order-paired (1→1, 2→2…); validate type compatibility per pair as they're drawn.
Navigation

Organize & navigate
  • Folders / projects Medium
    A project holds multiple diagrams. Builds on persistence.
  • Room templates new Medium
    Huddle room, classroom, streaming rig — fast starts from a pre-built diagram.
  • Revision history new Hard
    Natural with local files — version saved projects over time. ↳ SimpleWires charges for it.
  • Zones + labels Medium
    Color-coded, labeled regions (stage, rack, control room).
  • Zone-aware auto-arrange new Hard
    Auto-arrange / auto-layout should respect zones — keep items belonging to the same zone grouped together rather than scattering them across the canvas.
  • Move items together with their zone new Medium
    Dragging a zone moves its contained devices with it, so the zone and its items stay together as one unit.
  • Zone ⇄ tab — nested sub-diagrams new Epic
    A zone and a tab are two views of the same thing. Build out the control room as its own tab (a full sub-diagram), then drop that tab into the overall event diagram as a single zone block — and conversely promote any zone to its own tab. The inserted block exposes its boundary ports so you can draw connections between zones at the top level (control room → backstage, where backstage is itself another tab). Hierarchical diagrams: edit a room in detail, reference it as one unit elsewhere. ↳ Full design →
  • Deep cross-boundary grade validation new Hard
    Once diagrams nest, propagate a run's signal demand through a block's boundary port into the referenced tab's inner run — so an under-rated cable deep inside a control room still flags against the show format. Nesting v1 validates single-hop (host cable vs. the boundary port's mirrored grade); this is the deeper pass. ↳ decision 6 deferral from the zone⇄tab design.
  • Promote exposes a zone's dangling ports too new Easy
    Promoting a zone to a tab currently publishes a boundary port only per cable that crossed the zone edge, so a member's unconnected ports stay hidden (an unwired room can become an unwireable block). Embed already exposes all dangling ports — unify promote with it so the new block carries the room's full interface, then let the curate panel hide extras. ↳ small change to planPromoteZone; parity with embedTabAsBlock.
  • Boundary drift detection + refresh new Hard
    After a tab is embedded, editing the room can leave the block stale — a boundary port whose inner device/port was deleted, or whose connector/grade was re-spec'd. Flag drifted blocks (a derived-nodes layer, like displayEdges) and add a "refresh ports" action that re-mirrors/prunes the published boundary and updates every block referencing that room, switching boundary.rev from a constant to a content hash. ↳ Phase C of the zone⇄tab design; deferred at ship.
  • Curate a tab's published boundary ports new Medium
    Rename, hide, and reorder the boundary ports a tab exposes when embedded, so a room's public face stays intentional instead of the auto-derived full list. Hidden/renamed ports persist on the diagram's boundary. ↳ Phase C of the zone⇄tab design; deferred at ship.
  • Reorder tabs new Easy
    Drag tabs to change their order so related diagrams sit next to each other.
  • Save a zone/tab as a reusable build new Hard
    A built-out zone/tab — e.g. a custom flypack: ~15 devices and ~100 SDI cables wired internally — should be saveable as a reusable unit you can drop into future projects as a zone/tab, internal cabling and boundary ports intact. Open questions (explore later): export as a standalone file — reuse .sigpath or a dedicated subassembly format like .sigbuild? — and/or add a "custom builds" section to the library for multi-device zone/tab modules (local, with the community-share path as a stretch). Pairs with the zone ⇄ tab system and the custom item builder.
  • Headings / lists / formatted-text annotations Medium
  • Legend (cable-type color key) Easy
    Trivial once cable types exist.
  • Left-to-right mode · zoom-to-fit free · zoom-to-zone Medium
    fit is free via fitView; zoom-to-zone needs zones.
  • Rack elevation view new Epic
    A second view (19" front/rear) generated from the same devices. ↳ Stardraw's "one model → many views."
  • Light / dark mode free Easy
    React Flow colorMode; can slide earlier anytime.
  • Preferences pane — distance units (ft / m) new Medium
    Cable lengths are meters-only today. Stand up a small preferences/settings pane and put a distance-unit toggle in it (feet vs meters) — store internally in one canonical unit, convert on display and entry. The pane becomes the home for other app-wide settings later (theme, grid pitch, default cable type…).
  • Align grid pitch with port spacing — clean lines new Medium
    Make the snap grid and the port row spacing share a common unit so ports land on grid intersections and connections route as clean horizontal/orthogonal lines instead of slight diagonals.
  • Parallel cables stay independent — de-overlap or label new Hard
    Two cables running parallel currently merge into one visual line. Options to decide between: (a) auto-offset parallel runs so each reads as its own line; (b) a "combine + label" mode that bundles like cables and labels them where they merge and separate; (c) a per-diagram toggle between "every cable independent" and "combine & label." Likely the toggle, defaulting to independent. Phase 1 shipped: auto geometric de-overlap into stepped lanes (option a, always on); lane order flips with the bundle's travel direction so a downward fan nests instead of self-crossing. A final nudge pass (nudgeCollinearOverlaps) then fans apart any two runs of different cables still lying on the same line — scrambled patches and cross-device runs sharing a row — a few px with a small port hop, so nothing draws directly on top of anything else. And the cable ID now rides near BOTH ports, so a run reads at whichever end you trace from. Combine+label (b) and the per-diagram toggle (c) remain.
  • Manual cable lane nudge new Medium
    Pull a specific run out of a clump by nudging its jog. Shipped: a persisted jogOffset on the connection (round-tripped in save/load), nudged ◀ ▶ from the edge context bar; the first nudge starts from where the run currently sits (no jump), and a pinned run drops out of the crossing-minimizer so the rest route around it. "Auto" clears the override. Drag-to-move could layer on top — it'd set the same jogOffset.
  • Cables route around device boxes new Hard
    A cable whose straight run passes through a device box should route around it. Shipped: blocked runs are rerouted by a fewest-bends orthogonal A* over a sparse boundary grid (flow/obstacleRoute.ts), avoiding every device except their own two ends with a margin of clearance; clear runs stay on the fast default path + lane de-overlap. A whole bundle detouring one box is fanned apart (spreadDetourBundles) so the shared trunks read as separate nested lines, packed into the free space between obstacles. Each cluster picks the lane order (rank ascending vs descending) that yields fewer cable-over-cable crossings, so a planar fan detours with zero crossings and a scrambled one keeps only its unavoidable ones. A detour's descent is centred in the open corridor between the two devices (centerDetourVerticals) so it hugs neither the box it cleared (whose ports + outgoing cables line that edge) nor the target (whose input labels line its edge), and reads clearly between the two label columns. Zones and notes can opt in as obstacles too (per-element "Cables avoid" toggle, persisted; a region wrapping a cable's own endpoint is skipped).
  • Collapsible left & right sidebars new Easy
    Let the panels collapse to reclaim canvas width on smaller screens / dense diagrams.
  • Minimize / hide the minimap new Easy
    Let the user collapse or fully hide the React Flow minimap to reclaim corner space when it's not needed.
Output

  • Packlist lists PSUs with their device new Medium
    A device's power supply is its own line item in the packlist, named for the device it powers — e.g. a Netgear GS308v3 yields "PSU — Netgear GS308v3" so wall-warts / power bricks get packed too and are traceable to their gear.

  • Cable schedule + BOM export new Medium
    Port-to-port table (ID, type, length, from→to) and equipment-quantity list. ↳ no indie tool has this.
  • Printable cable labels new Medium
    Brother/Dymo-style label sheets. ↳ indie tools lack it.

Library

Community equipment database

Full design → Community Library Architecture. Two planes that never touch: a free static read path and a GitHub-PR write path. Git is the database — no Postgres mirror.

  • Grab / update from community library Medium
    Read-mostly local SQLite mirror, synced as a gzipped snapshot from a zero-egress CDN (GitHub Releases → R2). Offline-first; ETag/304 makes no-op updates free. Pull the whole snapshot each update — fine while the catalog is small.
  • Incremental sync — content-hash deltas new Hard
    Row contentHash + monotonic rev so updates ship as static deltas instead of re-downloading the full snapshot. Deferred until the catalog is large enough that whole-snapshot pulls (and unconstrained search) start to hurt — the seam is already left in the schema.
  • Seed the catalog — bulk import new Epic
    The real hard problem (not hosting): a CSV/JSON bulk-import path + a first pass of common gear, so the catalog is worth syncing on day one. Highest-risk workstream.
  • Community-updatable — account-free contribution Hard
    In-app Submit / Correct / Report → a stateless Cloudflare Worker validates and opens a GitHub PR via an app token. No GitHub account for users; no secrets shipped. Corrections fork the local copy and send a field-level diff. CC0-licensed.
  • In-app feedback box new Medium
    A simple "send feedback" panel in the UI — type a note, it posts to GitHub (as an Issue) via the same stateless Cloudflare Worker + app token as community contributions. No GitHub account for users. Reuse the Worker's validation, rate-limit, and pseudonymous install ID; attach app version / OS for context. Optionally label by type (bug / idea / praise).
  • Edit / correct a device new Medium
    Edit your own custom devices in place; editing a community/built-in device forks a personal copy (with provenance) so you can fix an error without deleting and starting over. The local half of corrections — pushing the fix upstream lights up with the contribution pipeline.
  • Define custom port/cable types (local connectors) new Hard
    Let a user define their own connector (port type = cable type) — id, label, color, group — merged into the registry so exotic gear is modelable locally. Precursor to submitting new connector types to the shared vocabulary (gated, minAppVersion-bumped). Requires refactoring the static CONNECTORS registry into a merged lookup.
  • Moderation & trust tiers new Hard
    GitHub PRs are the queue, diff viewer, and audit log. Pseudonymous install ID + rate-limit + blocklist (the only non-git state) tracks bad actors without logins. Auto-merge trivial fixes from trusted contributors; hold new devices.
  • North-star: "describe a room → starter diagram" new Epic
    XTEN-AV's AI frontier. Sits atop the community DB — prompt → starter diagram. Don't preclude it in the data model. ↳ the long-game capstone.
